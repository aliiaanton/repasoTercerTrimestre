Flashcards POO y UML

Haz clic en una tarjeta para ver la definición.

POO

Paradigma de programación basado en clases y objetos.

UML

Lenguaje de Modelado Unificado, estándar para visualizar, especificar y documentar sistemas software.

Clase

Plantilla para crear objetos, define atributos y métodos.

Objeto

Instancia de una clase, tiene estado (atributos) y comportamiento (métodos).

Encapsulación

Ocultar el estado interno de un objeto y requerir que toda interacción se realice a través de métodos. (Usar modificadores de acceso y getters/setters).

Abstracción

Exponer solo las características esenciales de un objeto, ocultando los detalles de implementación.

Herencia

Mecanismo por el cual una clase (derivada) adquiere atributos y métodos de otra clase (base). Relación "es-un-tipo-de".

Polimorfismo

Capacidad de objetos de diferentes clases para responder al mismo mensaje (método) de diferentes maneras.

Constructor

Método especial para inicializar un objeto cuando se crea. Mismo nombre que la clase, sin tipo de retorno.

Getter

Método público para obtener el valor de un atributo privado. (Ej: getDimensiones()).

Setter

Método público para establecer el valor de un atributo privado. (Ej: setDimensiones(int d)).

+

Vis. Pública

+

Indica que el miembro (atributo/método) es accesible desde cualquier clase.

-

Vis. Privada

-

Indica que el miembro solo es accesible desde dentro de la misma clase.

#

Vis. Protegida

#

Indica que el miembro es accesible desde la misma clase, subclases y clases del mismo paquete.

~

Vis. Paquete

~

Indica que el miembro es accesible solo desde clases dentro del mismo paquete (visibilidad por defecto en Java si no se especifica).

static

Modificador para miembros que pertenecen a la clase, no a las instancias. Se accede usando NombreClase.miembro.

final (var)

Indica que el valor de la variable no puede cambiar una vez inicializado (constante).

abstract (mét.)

Método sin implementación, debe ser implementado por subclases no abstractas.

abstract (clase)

Clase que no puede ser instanciada, puede contener métodos abstractos.

Interface

Contrato que define métodos (y constantes). Una clase `implementa` una interfaz.

Generalización

(Flecha sólida)

Relación UML de Herencia. Indica que la clase hija (origen flecha) "es-un-tipo-de" la clase padre (destino flecha).

Realización

(Flecha discontinua)

Relación UML de Implementación de interfaz. Indica que la clase (origen) cumple el contrato definido por la interfaz (destino).

Asociación

(Línea sólida)

Relación estructural general entre clases. Indica que los objetos están conectados o se conocen. Puede tener multiplicidad y navegabilidad.

Agregación

(Rombo hueco)

Tipo de asociación "tiene-un" o "parte-de". El ciclo de vida de la parte es independiente del todo (el rombo va en el lado del "todo").

Composición

(Rombo relleno)

Agregación fuerte "parte-de". El ciclo de vida de la parte depende del todo (la parte no existe sin el todo). El rombo va en el lado del "todo".

Dependencia

(Flecha discontinua)

Relación de uso. Una clase (origen) utiliza a otra (destino), p.ej., como parámetro de un método o variable local. Un cambio en el destino puede afectar al origen.
