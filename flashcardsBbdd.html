API JDBC

¿Qué es JDBC?

Java DataBase Connectivity. Es una API estándar de Java para conectar y ejecutar consultas con bases de datos.

Proporciona interfaces (Connection, Statement, ResultSet...).

Driver JDBC

Función

Implementación específica del fabricante de la BD (MySQL, Oracle, etc.) de las interfaces JDBC. Traduce las llamadas JDBC a comandos nativos de la BD.

Se carga (a menudo automáticamente) antes de usar.

Conexión (Connection)

¿Qué representa?

Representa una sesión con la base de datos. Se obtiene usando DriverManager.getConnection(url, user, password).

A través de ella se crean los Statements.

Ejecución de Sentencias

Objetos Principales

Se usan objetos Statement o PreparedStatement (creados desde Connection) para enviar instrucciones SQL a la BD.

Métodos: executeQuery() (para SELECT), executeUpdate() (para INSERT, UPDATE, DELETE).

Statement

Uso Básico

Interfaz para ejecutar sentencias SQL estáticas (sin parámetros). La SQL se pasa directamente a los métodos execute...().

Vulnerable a SQL Injection si se concatena entrada de usuario.

ResultSet

¿Qué contiene?

Representa el conjunto de resultados de una consulta (executeQuery()). Es una tabla de datos con un cursor.

Se itera con while(rs.next()). Se accede a datos con rs.getString("columna"), rs.getInt(indice), etc.

SQL Injection

¿Qué es y Peligro?

Ataque que inserta código SQL malicioso en consultas, a través de la entrada del usuario.

Peligro: Puede leer, modificar o borrar datos, obtener control del servidor.

Prevención: Usar PreparedStatement.

Sentencias Parametrizadas

PreparedStatement

Interfaz para ejecutar sentencias SQL precompiladas con parámetros (?).

Los valores se asignan con métodos setTipo(indice, valor) (ej: setString(1, "valor")).

Beneficios: Mejor rendimiento y previene SQL Injection.

Operaciones CRUD

¿Qué significa?

Acrónimo de las operaciones básicas en BBDD:

  • Create (Crear - INSERT)
  • Read (Leer - SELECT)
  • Update (Actualizar - UPDATE)
  • Delete (Borrar - DELETE)

Se ejecutan con executeQuery() o executeUpdate().

DAO (Objeto Acceso Datos)

Patrón de Diseño

Abstrae y encapsula el acceso a la fuente de datos (la lógica JDBC).

Proporciona una interfaz clara (ej: guardarCliente(), buscarClientePorId()) ocultando los detalles SQL y JDBC.

Mapeo Objeto-Relacional

Concepto Básico (ORM)

Técnica para convertir datos entre el modelo de objetos (clases Java) y el modelo relacional (tablas de BD).

Ej: Una clase Cliente se mapea a una tabla clientes, sus atributos a columnas.

(Frameworks como Hibernate/JPA lo automatizan).
